01 — Contexto

O Problema

A tarefa GWT-2392 foi aberta para investigar falhas nos follow-ups automáticos da Clara IA. Os dados do mês mostravam um padrão anômalo: a grande maioria dos leads recebia no máximo 2 follow-ups, e praticamente nenhum lead chegava aos FUPs 4, 5, 6 ou 7 — mesmo com o workflow configurado para enviar até 7 follow-ups por lead.

O workflow possui 3 blocos de execução paralelos, cada um com seu scheduler próprio:

02 — Diagnóstico

Falhas Identificadas

Falha Crítica — Block 2 nunca enviava FUPs 3 a 7

🔴 Crítico — Causa Raiz Principal

Linhas com respondeu = TRUE ausentes na tabela follow para cadências 3–7

O nó Consultar cadencia2 (Block 2) buscava o intervalo de tempo do próximo FUP filtrando cadencia = followup_count AND respondeu = TRUE. Porém, na tabela follow, apenas as cadências 1 e 2 possuíam linhas com respondeu = TRUE. As cadências 3, 4, 5 e 6 tinham apenas respondeu = NULL ou respondeu = FALSE. Resultado: o nó retornava 0 linhas para qualquer lead com followup_count > 2, e o lead era descartado silenciosamente da execução — sem erro, sem log, sem FUP enviado.

Falhas Secundárias

🟠 Infra — Intermitente

ETIMEDOUT no nó Consultar cadencia2 (Supabase)

Execução 80953518 (16/05) falhou com timeout de rede ao conectar no Supabase (IP 104.18.38.10:443). O nó não possuía "Retry on Fail" configurado. O AI Agent já tinha retry habilitado — os demais nós Supabase não.

🟠 Infra — Intermitente

Google Sheets HTTP 429 — Rate limit no Append row in sheet1

Quando o Block 2 processava múltiplos leads em sequência rápida, a API do Google Sheets retornava erro 429 (Too Many Requests). O intervalo configurado era de 5–10 segundos entre iterações do loop, insuficiente para o volume de requisições em horário de pico.

🔵 Atenção — Nome de coluna

Nó Consultar cadencia7 usa coluna Cadência (com acento)

O filtro desse nó usa keyName: "Cadência" (acento + maiúscula) enquanto a coluna real no Supabase é cadencia. O nó retorna 0 linhas silenciosamente. Verificar se esse nó está ativo e corrigir o nome da coluna se necessário.

03 — Correções Aplicadas

O Que Foi Corrigido

✅ Corrigido — Causa Raiz

Linhas respondeu = TRUE adicionadas na tabela follow para cadências 3–7

As linhas que estavam com respondeu = NULL (IDs 4, 5, 6, 7) foram atualizadas para respondeu = TRUE, seguindo o mesmo padrão das cadências 1 e 2. Com isso, o Consultar cadencia2 passa a encontrar os intervalos corretos para os FUPs 3, 4, 5, 6 e 7, desbloqueando o Block 2 completamente.

✅ Corrigido — Rate Limit Google Sheets

Intervalo entre iterações aumentado de 5–10s para 15–20s

O nó Intervalo (5 a 15s) foi atualizado para aguardar entre 15 e 20 segundos entre cada iteração do loop. Isso reduz a frequência de chamadas à API do Google Sheets e previne o erro 429 durante processamentos em lote no horário de pico.
